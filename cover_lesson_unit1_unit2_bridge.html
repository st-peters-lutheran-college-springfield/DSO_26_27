What Is SQL?

SQL (Structured Query Language) is the standard language used to create, read, update and delete data in a relational database. Every relational database engine — SQLite (used in your Unit 2 build), MySQL, PostgreSQL, SQL Server — understands the same core SQL commands. The syntax you practise below is what you'll use directly when you build your own devices/events database in Lesson 11.

SQLite vs MySQL: the SQL taught here works in both. A few small differences exist — SQLite uses AUTOINCREMENT, MySQL uses AUTO_INCREMENT; MySQL enforces column data types strictly, SQLite is more flexible. These differences don't affect anything in Unit 2, which uses SQLite.

Open the Execute SQL tab

Open dbMusic.db in DB Browser, then click the Execute SQL tab — this is where you'll type and run every command below.

1. CREATE TABLE (10 min)

Creates a new table — a structure to hold rows of data — by naming each column, its data type, and any constraints (such as a primary key).

CREATE TABLE table_name (
    column1 datatype constraints,
    column2 datatype constraints
);

Example — this is the actual statement that built the tblTracks table already in dbMusic.db:

CREATE TABLE tblTracks (
    TrackID INTEGER PRIMARY KEY AUTOINCREMENT,
    Title TEXT NOT NULL,
    Artist TEXT NOT NULL,
    Genre TEXT
);

Your turn: write a CREATE TABLE statement for a new table called tblPlaylists with columns PlaylistID (primary key), Name and Owner. Run it in DB Browser, then check the Database Structure tab to confirm it now exists.

2. INSERT INTO (10 min)

Adds a new row of data into a table.

INSERT INTO table_name (column1, column2) VALUES (value1, value2);

Example:

INSERT INTO tblPlaylists (PlaylistID, Name, Owner) VALUES (1, 'Road Trip', 'Sara Bibi');

Your turn: insert two more of your own playlists into tblPlaylists.

3. SELECT (15 min)

Retrieves data from a table. WHERE filters which rows come back; ORDER BY sorts the results.

SELECT column1, column2 FROM table_name WHERE condition ORDER BY column;

Examples using the tables already in dbMusic.db:

SELECT * FROM tblTracks WHERE Genre = 'Pop';

SELECT Firstname, Surname FROM tblMembers ORDER BY Surname;

Your turn:

  • Write a SELECT that returns every track in the "Rock" genre.
  • Write a SELECT that returns all your own playlists from tblPlaylists, ordered by Name.
  • Run this query against tblDownloads, then modify the year range yourself and re-run it: 
    SELECT * FROM tblDownloads WHERE date BETWEEN 2011 AND 2013;

4. UPDATE (10 min)

Changes existing data in a table. Always include a WHERE clause — without one, every row in the table is changed.

UPDATE table_name SET column = value WHERE condition;

Example:

UPDATE tblPlaylists SET Owner = 'Danny Judd' WHERE PlaylistID = 1;

Your turn: update the Name of one of your playlists to something new. Run SELECT * FROM tblPlaylists; afterwards to confirm the change.

5. DELETE (10 min)

Removes rows from a table. As with UPDATE, always include a WHERE clause — DELETE FROM table_name; with no WHERE removes every row.

DELETE FROM table_name WHERE condition;

Example:

DELETE FROM tblPlaylists WHERE PlaylistID = 2;

Your turn: delete one of the playlists you added earlier, then run SELECT * FROM tblPlaylists; to confirm only the remaining rows are left.
