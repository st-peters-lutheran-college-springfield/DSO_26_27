🔍 Lesson 2 — Identify your temperature & humidity sensor before you start

Look at the label on your sensor module. The wiring and one line of code differ between sensors — everything else stays the same.

DHT11

Label: "DHT11" · blue or white 3-pin Gravity module · common, reliable

Digital → D2 port (GPIO25)
DHT22 / AM2302

Label: "DHT22" or "AM2302" · 3-pin Gravity · more accurate than DHT11

Digital → D2 port (GPIO25)
DHT20

Label: "DHT20" · blue 4-pin Gravity module · I²C communication

I²C port · address 0x38

Not sure? Count connector pins: 3 pins = DHT11 or DHT22 (digital port). 4 pins = DHT20 (I²C port). The Lesson 2 tutorial shows all three.

Quick reference

🔌 Lesson 1 — PIR Wiring

  • IO Shield mounted on FireBeetle 2
  • PIR Gravity cable → D3 port on IO Shield
  • MicroPython: Pin(26, Pin.IN)
  • USB-C data cable → computer
  • WiFi: 2.4 GHz only (not 5 GHz)

🔌 Lesson 2 — Sensor Wiring

  • DHT11 or DHT22: 3-pin Gravity → D2 port (GPIO25)
  • DHT20: 4-pin Gravity → I²C port (address 0x38)
  • Set SENSOR = 'DHT11' / 'DHT22' / 'DHT20'
  • DHT11/22: built-in dht library — no install needed
  • DHT20: inline driver in code — no library install

💡 Programming Concepts Checklist

  • Selection — if/elif/else in evaluate_rest()
  • Iteration — while True server loop
  • User input — browser HTTP GET request
  • Data output — HTML dashboard + Shell print
  • Functions — read / evaluate / build / serve
  • Error handling — try/except around sensor reads

🚨 Troubleshooting

  • COM port error — use data cable, not charge-only
  • WiFi fails — 2.4 GHz? correct SSID/password?
  • Page won't load — same network? exact IP? try incognito
  • Wrong sensor — check SENSOR = line in main.py
  • DHT reads None — D2 port for DHT11/22, I²C port for DHT20
  • Saved to wrong place — "MicroPython device" not "This computer"
