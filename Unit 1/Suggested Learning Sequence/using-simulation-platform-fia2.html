Why Simulate First?

The EEDU Kit uses real hardware that can be damaged, misconfigured, or unavailable before a class session. Wokwi lets you write and test the same logic before connecting physical components, which reduces setup time and gives you clearer evidence of your problem-solving process.

What Wokwi Gives You
  • Virtual ESP32 simulation in-browser
  • Sensor interaction without hardware setup delays
  • MicroPython starter support for quick testing
  • Easy sharing of project links and screenshots
Simulation Limitations
  • Wokwi uses a generic ESP32 board, not the exact classroom board
  • Sensor values are simulated, not real-world calibrated data
  • Physical wiring constraints may still require later adjustment

Rust Pathway Prerequisite

Rust pathway prerequisite Before using the Rust simulation pathway in Wokwi, complete the Rust on ESP32 — Flipped Classroom Hub. This lesson covers the Rust toolchain, ESP32 project structure, core syntax, GPIO basics, and the setup required before Rust simulation will work successfully.
Suggested sequence
  1. Complete the Rust on ESP32 — Flipped Classroom Hub.
  2. Set up the Rust toolchain and build a simple project locally.
  3. Return to this simulation lesson to apply the FIA2 workflow in Wokwi.
