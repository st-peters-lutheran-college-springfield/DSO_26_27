Part 2 — The five useability principles & Daniel's dashboard

♿ Accessibility

Can everyone use the interface, including people with disabilities or impairments?

→ Daniel has colour-vision deficiency. His dashboard uses ✓/● symbols and text labels — not colour alone.

✅ Effectiveness

Does the interface help users achieve their goal accurately and completely?

→ Daniel's goal: know if room is calm. Dashboard shows "ROOM IS CALM" — one glance, goal achieved.

🛡️ Safety

Does the design prevent user errors and protect them from dangerous mistakes?

→ Dashboard is read-only — Daniel can't accidentally change settings when half-asleep.

🔧 Utility

Does the interface provide the right functions for what the user actually needs?

→ Dashboard shows motion status and calm timer — exactly what Daniel needs, nothing more.

📖 Learnability

How quickly can a new user understand how to use the interface?

→ No buttons, no menus, no login. Daniel can use it with zero instruction — maximum learnability.
📓 Journal Task 2 — ~8 minutes

In your journal, write one sentence for each of the five useability principles explaining how your PIR dashboard either meets or fails to meet that principle.

Part 3 — Visual design principles (CARP) in the QCAA syllabus

The QCAA syllabus lists these visual design principles for Unit 1 Topic 2: balance, contrast, proximity, harmony, alignment, repetition and hierarchy. The acronym CARP (Contrast, Alignment, Repetition, Proximity) covers four of the most important ones for screen design.

C — Contrast

Make different things look different. If two things aren't the same, make them very different — not just slightly different.

  • Dark background (#111) vs near-white text (#eee)
  • Large symbol (✓) vs small label text
  • Solid / dashed / dotted borders to signal state

Syllabus link: contrast, hierarchy, tone

A — Alignment

Every element should have a visual connection to something else on the page. Nothing should be placed arbitrarily.

  • All text centred — consistent axis throughout
  • Reading tiles aligned in a grid
  • Progress bar full-width — edge-to-edge alignment

Syllabus link: alignment, balance

R — Repetition

Repeat visual elements throughout the design. This creates consistency and strengthens unity.

  • Same monospace font everywhere
  • Same tile style for every reading card
  • Same border-radius on all containers

Syllabus link: repetition, harmony

P — Proximity

Group related items together. Items that are related should be close; unrelated items should be separated.

  • Symbol + status label + message grouped in one card
  • Progress bar separated from status card
  • Reading tiles separate from main status area

Syllabus link: proximity, hierarchy

🎨 Remaining syllabus principles — also in your design

Balance

Visual weight is distributed evenly. Daniel's dashboard is vertically centred and full-width — no heavy left or right side.

Harmony

All visual elements feel like they belong together. Using a single font family, monochrome palette, and consistent spacing creates harmony.

Hierarchy

The most important information (status symbol + label) is largest and at the top. Secondary data (readings) is smaller below.

Part 4 — Your 2-minute assessment video

🎬 What the assessment requires

From the FIA2 task sheet: "Visual and spoken video: up to 2 minutes, demonstrating the functionality of the user interface and data and programmed components."

✅ Must include

  • Your dashboard running live in a browser
  • At least one sensor changing the status (wave hand at PIR)
  • Your spoken explanation of what is happening
  • Reference to Daniel's needs and your design decisions
  • At least one UX or visual design principle named and explained

🚫 Do not include

  • More than 2 minutes of footage
  • Just showing code — the video must show the running interface
  • Silent screen recording — you must speak
  • Background music that drowns out your voice
  • Other students' names or faces
