Pico W Setup — Mac M4 & DHT22 + PIR

Your Mac M4 can't connect to the FireBeetle 2 ESP32-E because of a USB driver issue on Apple Silicon. The Raspberry Pi Pico W uses a completely different USB system that works on Mac M4 with no drivers needed. Your DHT22 and PIR sensors from the kit connect directly to the Pico W with jumper wires. The MicroPython code is almost identical.

🍎 Mac M4 — no drivers needed 🐍 MicroPython v1.24+ — same language 🌡️ DHT22 → GP15 👁 PIR → GP14 📡 Built-in 2.4 GHz WiFi ⏱ ~30 minutes setup
FIA2 Project Hub › Pico W Setup — Mac M4
Why your Mac M4 couldn't connect to the FireBeetle 2

❌ FireBeetle 2 ESP32-E (old board)

  • Uses a CH340G USB-to-serial chip to connect to computers
  • CH340G requires a third-party driver to be installed
  • That driver has known compatibility issues with macOS on Apple Silicon (M1/M2/M3/M4)
  • Result: Mac can't see the board in Thonny

✅ Raspberry Pi Pico W (your new board)

  • Uses a built-in USB interface on the RP2040 chip — no external chip
  • No driver installation needed on any OS — Mac, Windows, Linux
  • Mac M4 recognises it immediately as a serial device (/dev/cu.usbmodem…)
  • Same MicroPython language — almost all your code is identical
What you need

🖥️ Mac M4

  • Thonny IDE (ARM64 build for Apple Silicon)
  • Safari, Chrome, or Firefox
  • OneNote app or browser
  • No drivers to install

🔬 Raspberry Pi Pico W

  • RP2040 chip — dual-core 133 MHz
  • Built-in 2.4 GHz WiFi (same frequency as ESP32)
  • Micro-USB port only
  • MicroPython v1.24+ — latest stable

🧰 From Your EEDU Kit

  • Gravity DHT22 temperature & humidity sensor
  • Gravity PIR motion sensor
  • USB data cable (USB-C to micro-USB or USB-A)
  • 6× female-to-male Dupont jumper wires

⚠️ Not Needed for Pico W

  • ❌ Gravity IO Shield (FireBeetle 2 only)
  • ❌ Gravity I²C Hub
  • ❌ CH340 driver
  • ❌ Gravity-to-Gravity cables (use jumper wires instead)
🔌

About the Gravity connectors

The DHT22 and PIR sensors use Gravity connectors designed for the IO Shield. Without the IO Shield, you need to connect directly to the Pico W pins using female-to-male Dupont jumper wires. The Gravity cable has three wires: black = GND, red = VCC, yellow/white = SIG (data signal). If your Gravity cable has female ends on both sides, plug one female end into the sensor and use the individual female connectors on the other end to connect to the Pico W male pins.

Setup steps — do these in order
1
⏱ 3 minutes

Download the Pico W MicroPython firmware

You must download the firmware for the Pico W specifically — the standard Pico firmware will not work (it lacks the WiFi driver).

  1. Go to micropython.org/download/RPI_PICO_W/ ↗
  2. Under "Releases", click the first .uf2 link — this is the latest stable version (v1.28.0 or newer)
  3. Save the .uf2 file to your Downloads folder
⚠️

Do not download the plain Pico or Pico 2 firmware — only RPI_PICO_W. Getting the wrong firmware is the most common error — the file must disappear from the drive after copying. If it stays there, you have the wrong firmware.

2
⏱ 3 minutes

Flash MicroPython onto your Pico W

This only needs to be done once. The firmware stays on the board permanently.

  1. Locate the small BOOTSEL button on top of the Pico W — it is the only button on the board
  2. Hold the BOOTSEL button down with your finger
  3. While still holding BOOTSEL, plug the USB cable into your Mac — keep holding for 2 more seconds, then release
  4. A drive called RPI-RP2 should appear on your Mac desktop (in Finder)
  5. Drag and drop the .uf2 file you downloaded in Step 1 into the RPI-RP2 drive
  6. The drive will disappear automatically — this means it worked ✅
  7. The Pico W reboots with MicroPython installed
✅

If the drive disappears after copying: firmware installed successfully. If the drive stays visible and the file is still there, you downloaded the wrong firmware — go back to Step 1 and make sure you chose RPI_PICO_W.

3
⏱ 3 minutes

Install or check Thonny on your Mac M4

Thonny must be the Apple Silicon (ARM64) version — the Intel version may not work correctly on Mac M4.

  1. Go to thonny.org ↗
  2. Click the Mac download button — ensure it says Apple Silicon or ARM64 in the filename (e.g. thonny-x.x.x-arm64.pkg)
  3. Open the downloaded .pkg file and follow the install steps
  4. If you already have Thonny, check: Thonny → About Thonny — version should be 4.1 or newer
ℹ️

If macOS shows "cannot be opened because the developer cannot be verified", go to System Settings → Privacy & Security and click "Open Anyway" next to the Thonny message.

4
⏱ 2 minutes

Connect Thonny to the Pico W

Plug in your Pico W (without holding BOOTSEL this time — just plug in normally).

  1. Open Thonny
  2. Click Run → Configure interpreter…
  3. From the dropdown, select MicroPython (Raspberry Pi Pico)
  4. Under Port, select the option that contains usbmodem — e.g. /dev/cu.usbmodem1101
  5. Click OK

You should see MicroPython v1.24.x; Raspberry Pi PicoW with RP2040 and a >>> prompt in the Thonny Shell at the bottom. ✅

⚠️

If no port appears in the list: Unplug and replug the USB cable. Try a different USB port on your Mac. Make sure you are NOT holding BOOTSEL when plugging in — that puts it in firmware mode, not serial mode. If still nothing, restart Thonny.

5
⏱ 8 minutes

⚡ Power notes
Both sensors run at 3.3V — use the 3V3 OUT pin, not VBUS (5V). The Gravity PIR operates correctly at 3.3V. Two sensors share the same 3V3 pin — use a breadboard or Y-splitter.
SensorWire ColourPico W PinPhysical Pin #Notes

Red 3V3 OUT Pin 36 3.3V power — share with DHT22 via breadboard

💡

Sharing the 3V3 pin between two sensors

Both sensors need 3.3V power but the Pico W has only one labelled 3V3 OUT pin (pin 36). Use a mini breadboard from the kit to connect both red wires to the same 3.3V rail, or use a female-to-two-female splitter wire. Alternatively, GP16's 3V3 pin (pin 36) is the same as pin 36 — check your specific Pico board for multiple 3V3 pins, as some boards have two.

⏱ 5 minutes

Write boot.py — WiFi connection (identical to other boards)

The WiFi code for Pico W is exactly the same as the ESP32 version. Just change your network details.

boot.py — File → Save as → MicroPython device → boot.py MicroPython · Pico W 
# boot.py — Raspberry Pi Pico W
# Runs on startup before main.py
# WiFi code is identical to ESP32 version

import network, time

WIFI_NAME     = 'YourNetworkName'   # ← change this
WIFI_PASSWORD = 'YourPassword'       # ← change this

wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect(WIFI_NAME, WIFI_PASSWORD)

timeout = 15
while not wlan.isconnected() and timeout > 0:
    time.sleep(1)
    timeout -= 1

if wlan.isconnected():
    print('Connected! Open in browser:')
    print('  http://' + wlan.ifconfig()[0])
else:
    print('WiFi failed — check name and password (2.4 GHz only)')
  1. File → New in Thonny
  2. Paste the code above and change your WiFi details
  3. File → Save as… → MicroPython device → boot.py
  4. Press the small RUN button (▶) in Thonny or press the RST/RUN button on the Pico
  5. Look for your IP address in the Shell, e.g. http://192.168.1.42 — write it down

⏱ 10 minutes

Write main.py — DHT22 + PIR + web server

This code covers both Lesson 1 (PIR) and Lesson 2 (temperature & humidity) combined, so you can catch up to the rest of the class. The only differences from the class code are the pin numbers: GP15 for DHT22 and GP14 for PIR.

main.py — save to MicroPython device MicroPython · Pico W · DHT22 GP15 · PIR GP14 
# ─────────────────────────────────────────────────────────────
# main.py  |  Raspberry Pi Pico W  |  MicroPython v1.28
# FIA2 Project — Rest Environment Monitor
# Sensors: DHT22 → GP15  |  PIR → GP14
#
# PIN DIFFERENCES from class FireBeetle 2 code:
#   FireBeetle 2: DHT11/22 → GPIO25 (D2)  PIR → GPIO26 (D3)
#   Pico W:     DHT22    → GP15         PIR → GP14
#   Everything else is IDENTICAL
# ─────────────────────────────────────────────────────────────

from machine import Pin
import socket, time, dht


# ════════════════════════════════════════════════════════════
# SECTION 1 — Sensor Setup
# DHT22 on GP15 (physical pin 20)
# PIR   on GP14 (physical pin 19)
# ════════════════════════════════════════════════════════════

# DHT22 — digital single-wire protocol
# Built-in MicroPython dht library works identically on Pico W
_dht = dht.DHT22(Pin(15))           # GP15

def read_sensor():
    """Read temperature (°C) and humidity (%) from DHT22."""
    try:
        _dht.measure()
        return float(_dht.temperature()), float(_dht.humidity())
    except OSError:
        return None, None

# PIR motion sensor — digital input
pir            = Pin(14, Pin.IN)     # GP14
last_motion_ms = time.ticks_add(time.ticks_ms(), -61000)
CALM_SECONDS   = 60
WARMUP_MS      = 30000
startup_ms     = time.ticks_ms()

def check_pir():
    global last_motion_ms
    if pir.value() == 1: last_motion_ms = time.ticks_ms()
    return pir.value() == 1

def seconds_since_motion():
    return time.ticks_diff(time.ticks_ms(), last_motion_ms) // 1000


# ════════════════════════════════════════════════════════════
# SECTION 2 — 3-Condition Rest Algorithm (same as class code)
# Condition A: PIR calm for ≥ 60 s
# Condition B: Temperature 16–22 °C
# Condition C: Humidity < 70 %
# ════════════════════════════════════════════════════════════

TEMP_MIN, TEMP_MAX, HUM_MAX = 16.0, 22.0, 70.0

def evaluate_rest(temp, hum):
    motion_now = check_pir()
    secs       = seconds_since_motion()
    warmup     = time.ticks_diff(time.ticks_ms(), startup_ms) < WARMUP_MS
    motion_ok  = (not motion_now) and (secs >= CALM_SECONDS) and (not warmup)
    temp_ok    = temp is not None and TEMP_MIN <= temp <= TEMP_MAX
    hum_ok     = hum  is not None and hum <= HUM_MAX
    score      = (1 if motion_ok else 0) + (1 if temp_ok else 0) + (1 if hum_ok else 0)
    secs_str   = '{}m {}s'.format(secs//60, secs%60) if secs >= 60 else '{}s'.format(secs)

    if   warmup:       status, advice = 'STARTING UP',       'Sensors warming up — wait 30 s.'
    elif score == 3:  status, advice = 'SUITABLE FOR REST', 'All conditions met. Room is ready.'
    elif score == 0:  status, advice = 'NOT SUITABLE',      'Motion, temp and humidity all need attention.'
    else:
        issues = []
        if not motion_ok: issues.append('motion {}' .format('now' if motion_now else secs_str + ' ago'))
        if not temp_ok and temp is not None: issues.append('temp {:.1f}C'.format(temp))
        if not hum_ok  and hum  is not None: issues.append('hum {:.0f}%'.format(hum))
        status = 'NEEDS ADJUSTMENT'
        advice = 'Fix: ' + ', '.join(issues)

    return status, advice, score, motion_ok, temp_ok, hum_ok, secs


# ════════════════════════════════════════════════════════════
# SECTION 3 — HTML Dashboard (identical to class version)
# ════════════════════════════════════════════════════════════

def build_page(temp, hum, status, advice, score, m_ok, t_ok, h_ok, secs):
    t_s = '{:.1f}°C'.format(temp) if temp is not None else '---'
    h_s = '{:.0f}%'.format(hum)  if hum  is not None else '---'
    syms  = {3:'\u2713', 2:'~', 1:'!', 0:'\u2717'}
    brds  = {3:'border:3px solid #93c5fd', 2:'border:3px dashed #60a5fa',
             1:'border:3px dotted #3b82f6', 0:'border:3px dotted #1d4ed8'}
    sym, brd = syms.get(score, '?'), brds.get(score, 'border:3px dotted #1d4ed8')
    filled = min(10, secs * 10 // CALM_SECONDS)
    bar    = '\u2588' * filled + '\u2591' * (10 - filled)
    mb, tb, hb = ('[OK]' if m_ok else '[--]'), ('[OK]' if t_ok else '[--]'), ('[OK]' if h_ok else '[--]')

    return """<!DOCTYPE html>
<html lang="en"><head>

<meta http-equiv="refresh" content="5"><title>Rest Monitor</title>

     display:flex;flex-direction:column;align-items:center;
     min-height:100vh;padding:18px 12px;gap:12px}}
h1{{font-size:.72rem;letter-spacing:4px;color:#1e3a8a;text-transform:uppercase}}
.card{{background:#0f2044;border-radius:10px;padding:20px 22px;
       text-align:center;width:100%;max-width:420px}}
.sym{{font-size:3.8rem;line-height:1.1;color:#93c5fd}}

<body>
  <h1>&#9632; REST ENVIRONMENT MONITOR</h1>

        t=t_s, h=h_s, mb=mb, tb=tb, hb=hb,
        mc='ok' if m_ok else 'fail',
        tc='ok' if t_ok else 'fail',
        hc='ok' if h_ok else 'fail'

# ════════════════════════════════════════════════════════════
# SECTION 4 — Web Server (identical to class version)
# ════════════════════════════════════════════════════════════

srv = socket.socket()
srv.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
srv.bind(('', 80)); srv.listen(3)
print('Server ready | DHT22:GP15 | PIR:GP14 | Pico W')

while True:
    try:
        conn, _ = srv.accept()
        conn.recv(1024)
        temp, hum = read_sensor()
        status, advice, score, m_ok, t_ok, h_ok, secs = evaluate_rest(temp, hum)
        print('T:{} H:{} Calm:{}s → {}'.format(temp, hum, secs, status))
        html = build_page(temp, hum, status, advice, score, m_ok, t_ok, h_ok, secs)
        conn.sendall(('HTTP/1.1 200 OK\r\nContent-Type: text/html\r\nConnection: close\r\n\r\n' + html).encode())
        conn.close()
        time.sleep(2)     # DHT22 needs 2 s between reads
    except Exception as e:
        print('Error:', e)
        try: conn.close()
        except: pass
