PIR Motion Sensor Rest Monitor

🚀 This is your starting point

The code in this lesson is deliberately simple — it meets the basic requirements and gives you something working to build on. Once it's running, you'll extend it by adjusting thresholds, adding sensors, and improving the dashboard design to make it your own solution for Daniel.

🎯 What you will build

A MicroPython program that:

  • Reads the PIR motion sensor every time a browser connects
  • Tracks how long ago motion was last detected
  • Decides one of three states: CALM / SETTLING / DETECTED
  • Serves a high-contrast dashboard — no interaction needed

👤 Daniel's requirement

He glances at a mounted tablet. He needs:

  • A clear YES/NO answer — not raw sensor data
  • High contrast — mild colour-vision deficiency
  • No interaction — passive display, always live
  • Plain English — "Room is calm" not "0 counts"

🧰 What you need

  • FireBeetle 2 ESP32-E (MicroPython flashed)
  • Gravity IO Shield (on top of board)
  • Gravity PIR Motion Sensor + 3-pin cable
  • USB-C data cable
  • Computer with Thonny IDE installed
  • WiFi network (2.4 GHz)

import network, time

WIFI_NAME     = 'YourNetworkName'   # ← change this
WIFI_PASSWORD = 'YourPassword'       # ← change this

wlan = network.WLAN(network.STA_IF)
wlan.active(True)
wlan.connect(WIFI_NAME, WIFI_PASSWORD)

# ITERATION — wait up to 15 seconds for WiFi
timeout = 15
while not wlan.isconnected() and timeout > 0:
    time.sleep(1)
    timeout -= 1

# SELECTION — did we connect?
if wlan.isconnected():
    print('Connected! Open this in your browser:')
    print('  http://' + wlan.ifconfig()[0])
else:
    print('WiFi failed — check name and password')

# main.py — PIR Motion Sensor Rest Monitor
# FireBeetle 2 ESP32-E | EEDU IoT Kit | MicroPython
# STARTING POINT — extend this code to build your own solution

from machine import Pin
import socket, time


# ── SECTION 1: Hardware ───────────────────────────────────────
# PIR Gravity cable → D3 port on IO Shield → GPIO26
# pir.value() returns: 1 = motion detected, 0 = no motion

pir = Pin(26, Pin.IN)

# ticks_ms() counts milliseconds since the board powered on.
# We record WHEN motion last happened so we can ask:
# "how many seconds have passed since then?"
last_motion_ms = time.ticks_add(time.ticks_ms(), -61000)  # start as if 61 s ago

CALM_SECONDS = 60   # room must be still for this many seconds
                     # try changing to 10 during testing, reset to 60 after


# ── SECTION 2: Algorithm ─────────────────────────────────────
# This function answers Daniel's question:
# "Is my room calm enough to rest right now?"
#
# Returns: state (str), symbol (str), message (str), seconds (int)

def get_status():
    global last_motion_ms

    # Check PIR — update timer if motion is detected right now
    if pir.value() == 1:
        last_motion_ms = time.ticks_ms()

    # How many whole seconds since motion was last detected?
    elapsed_ms = time.ticks_diff(time.ticks_ms(), last_motion_ms)
    seconds    = elapsed_ms // 1000

    # SELECTION — 3-state decision based on PIR + elapsed time
    if pir.value() == 1:
        # Motion is happening RIGHT NOW
        return 'DETECTED', '\u25cf', 'Movement detected now. Wait for calm.', seconds

    elif seconds < CALM_SECONDS:
        # No motion right now but not enough time has passed yet
        remaining = CALM_SECONDS - seconds
        msg = 'Last movement {}s ago. Need {}s more of calm.'.format(seconds, remaining)
        return 'SETTLING', '~', msg, seconds

    else:
        # Room has been undisturbed long enough — suitable for rest
        mins    = seconds // 60
        sec_rem = seconds % 60
        duration = '{}m {}s'.format(mins, sec_rem) if mins > 0 else '{}s'.format(seconds)
        return 'CALM', '\u2713', 'No movement for {}. Room is ready.'.format(duration), seconds


# ── SECTION 3: Build the HTML page ───────────────────────────
# Generates the page Daniel sees on his tablet.
# Design: dark background, high contrast, symbols not colours.

def build_page(state, symbol, message, seconds):

    # Text progress bar — fills as the room stays calm
    filled = min(10, seconds * 10 // CALM_SECONDS)
    bar    = '\u2588' * filled + '\u2591' * (10 - filled)   # █ and ░
    bar_sub = '{}s / {}s threshold'.format(min(seconds, CALM_SECONDS), CALM_SECONDS)

    # Border STYLE changes with state — accessible without colour
    if   state == 'CALM':     border = 'border:3px solid #ddd'
    elif state == 'SETTLING': border = 'border:3px dashed #aaa'
    else:                      border = 'border:3px dotted #777'

    return """<!DOCTYPE html>
<html lang="en"><head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width,initial-scale=1">
<meta http-equiv="refresh" content="5">
<title>Rest Monitor</title>
<style>
  body{{background:#111;color:#eee;font-family:monospace;
       display:flex;flex-direction:column;align-items:center;
       min-height:100vh;padding:20px 14px;gap:14px}}
  h1{{font-size:.75rem;letter-spacing:4px;color:#444;text-transform:uppercase}}
  .card{{background:#1c1c1c;border-radius:10px;
         padding:22px 24px;text-align:center;width:100%;max-width:380px}}
  .sym{{font-size:4rem;line-height:1.1}}
  .state{{font-size:1.2rem;font-weight:bold;letter-spacing:1px;margin:6px 0 4px}}
  .msg{{font-size:.84rem;color:#666;line-height:1.55}}
  .bwrap{{background:#141414;border:1px solid #1e1e1e;border-radius:7px;
          padding:10px 12px;width:100%;max-width:380px}}
  .bl{{font-size:.56rem;letter-spacing:2px;color:#333;text-transform:uppercase}}
  .bar{{font-size:1rem;letter-spacing:4px;color:#888;margin:4px 0 2px}}
  .bs{{font-size:.6rem;color:#2e2e2e}}
  footer{{font-size:.6rem;color:#252525;text-align:center}}
</style></head>
<body>
  <h1>&#9632; REST ENVIRONMENT MONITOR</h1>
  <div class="card" style="{border};background:#1c1c1c;border-radius:10px;
                            padding:22px 24px;text-align:center;
                            width:100%;max-width:380px">
    <div class="sym">{symbol}</div>
    <div class="state">{state}</div>
    <div class="msg">{message}</div>
  </div>
  <div class="bwrap">
    <div class="bl">CALM DURATION</div>
    <div class="bar">{bar}</div>
    <div class="bs">{bar_sub}</div>
  </div>
  <footer>Updates every 5 s &mdash; no interaction needed &middot; FireBeetle 2 &middot; PIR</footer>
</body></html>""".format(
        border=border, symbol=symbol, state=state,
        message=message, bar=bar, bar_sub=bar_sub
    )


# ── SECTION 4: Web Server ─────────────────────────────────────
# ITERATION: while True loop — wait for a browser to connect,
# send the HTML page, then loop back and wait again.

server = socket.socket()
server.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
server.bind(('', 80))   # port 80 = standard HTTP
server.listen(3)
print('Server ready — open your IP address in a browser')

while True:                 # ITERATION — run forever
    try:
        conn, _ = server.accept()        # wait for browser to connect
        conn.recv(1024)                  # read the request (we ignore it)

        state, symbol, message, seconds = get_status()

        print('[{}] {}s since motion'.format(state, seconds))  # debug

        page     = build_page(state, symbol, message, seconds)
        response = ('HTTP/1.1 200 OK\r\n'
                    'Content-Type: text/html\r\n'
                    'Connection: close\r\n\r\n' + page)
        conn.sendall(response.encode())
        conn.close()

    except Exception as e:
        print('Error:', e)
        try: conn.close()
        except: pass

🚨 Troubleshooting — if something's not working
  • COM port not found in Thonny — try a different USB cable (must be a data cable, not charge-only)
  • WiFi connection fails — check SSID and password are exactly right. Network must be 2.4 GHz. Try your phone's hotspot.
  • Page won't load in browser — are you on the same network as the FireBeetle 2? Did you type the IP correctly from the Shell?
  • PIR fires immediately on startup — this is normal warm-up behaviour. Wait 30 seconds.
  • "File not found" or code not running — did you save to "MicroPython device" not "This computer"? Check the file panel in Thonny.
