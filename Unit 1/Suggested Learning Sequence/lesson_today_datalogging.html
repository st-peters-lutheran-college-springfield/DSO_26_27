⏰ What is an RTC?

RTC = Real-Time Clock. The SD2405 is a small I²C chip that keeps track of the current date and time — even when the board is powered off (it has its own battery).

  • I²C address: 0x32
  • Tracks: seconds, minutes, hours, day, month, year
  • Gravity 4-pin connector → I²C port on IO Shield

👤 Why this helps Daniel

  • Daniel arrives home at different times — logged data shows whether conditions are typically better at 6am vs 2pm
  • Patterns reveal if the room is usually warm during afternoon, helping Daniel plan when to use a fan
  • Logging evidence of poor conditions justifies future improvements (e.g. better insulation)
  • Time-stamped data supports the FIA2 data repository requirement

_i = I2C(0, scl=Pin(22), sda=Pin(21), freq=100000)
_A = 0x32

# ── SET THESE TO TODAY'S DATE AND TIME ─────────────────────
# Format:  sec, min, hour, weekday(1=Mon), day, month, year
sec, mn, hr, wd, day, mon, yr = 0, 30, 9, 1, 30, 5, 25
# ────────────────────────────────────────────────────────────

def bcd(v): return ((v // 10) << 4) | (v % 10)

# Enable write, set time, disable write
_i.writeto(_A, bytes([0x10, 0x80]))
_i.writeto(_A, bytes([0x00, bcd(sec), bcd(mn), bcd(hr), bcd(wd), bcd(day), bcd(mon), bcd(yr)]))
_i.writeto(_A, bytes([0x10, 0x00]))
print('RTC time set!')

# ════════════════════════════════════════════════════════════
# SECTION 1 — RTC SD2405  (I²C address 0x32)
# Reads the current date and time as a formatted string.
# BCD = Binary Coded Decimal — the SD2405 stores time in this
# format. bcd_to_int() converts each byte back to a number.
# ════════════════════════════════════════════════════════════

_rtc_bus = I2C(0, scl=Pin(22), sda=Pin(21), freq=100000)
RTC_ADDR = 0x32

def bcd_to_int(b): return (b >> 4) * 10 + (b & 0x0F)

def read_rtc():
    """
    Read date and time from RTC SD2405.
    Returns: formatted string 'HH:MM:SS DD/MM/YY'
             or 'RTC error' if the sensor fails.
    """
    try:
        _rtc_bus.writeto(RTC_ADDR, bytes([0x00]))       # point to register 0
        d = _rtc_bus.readfrom(RTC_ADDR, 7)             # read 7 bytes
        sec  = bcd_to_int(d[0] & 0x7F)
        mn   = bcd_to_int(d[1] & 0x7F)
        hr   = bcd_to_int(d[2] & 0x3F)
        day  = bcd_to_int(d[4] & 0x3F)
        mon  = bcd_to_int(d[5] & 0x1F)
        yr   = bcd_to_int(d[6])
        return '{:02d}:{:02d}:{:02d} {:02d}/{:02d}/{:02d}'.format(hr, mn, sec, day, mon, yr)
    except Exception as e:
        print('RTC error:', e)
        return 'RTC error'

# SECTION 3 — PIR (unchanged from Lessons 1 & 2)

pir = Pin(26, Pin.IN)
last_motion_ms = time.ticks_add(time.ticks_ms(), -61000)
CALM_SECONDS   = 60
WARMUP_MS      = 30000
startup_ms     = time.ticks_ms()

def check_pir():
    global last_motion_ms
    if pir.value() == 1: last_motion_ms = time.ticks_ms()
    return pir.value() == 1

def seconds_since_motion():
    return time.ticks_diff(time.ticks_ms(), last_motion_ms) // 1000


# ════════════════════════════════════════════════════════════
# SECTION 4 — Algorithm (same 3-condition logic as Lesson 2)
# ════════════════════════════════════════════════════════════

TEMP_MIN, TEMP_MAX, HUM_MAX = 16.0, 22.0, 70.0

def evaluate_rest(temp, hum):
    motion_now = check_pir()
    secs       = seconds_since_motion()
    warmup     = time.ticks_diff(time.ticks_ms(), startup_ms) < WARMUP_MS
    motion_ok  = (not motion_now) and (secs >= CALM_SECONDS) and (not warmup)
    temp_ok    = temp is not None and TEMP_MIN <= temp <= TEMP_MAX
    hum_ok     = hum  is not None and hum <= HUM_MAX
    score      = (1 if motion_ok else 0) + (1 if temp_ok else 0) + (1 if hum_ok else 0)
    if warmup:
        status, advice = 'STARTING UP', 'Sensors warming up...'
    elif score == 3:
        status, advice = 'SUITABLE FOR REST', 'All conditions met. Room is ready.'
    elif score == 0:
        status, advice = 'NOT SUITABLE', 'Motion, temp and humidity all need attention.'
    else:
        issues = []
        if not motion_ok: issues.append('motion')
        if not temp_ok and temp is not None: issues.append('temp {:.1f}C'.format(temp))
        if not hum_ok  and hum  is not None: issues.append('hum {:.0f}%'.format(hum))
        status = 'NEEDS ADJUSTMENT'
        advice = 'Fix: ' + ', '.join(issues)
    return status, advice, score, motion_ok, temp_ok, hum_ok, secs


# ════════════════════════════════════════════════════════════
# SECTION 5 — Data Log  (the data repository)
#
# 'log' is a Python list that stores dictionaries.
# Each dictionary = one reading with a timestamp.
# ITERATION: when the log reaches MAX_LOG entries,
# the oldest entry (index 0) is removed to make room.
# ════════════════════════════════════════════════════════════

log = []  # in-memory data repository — cleared on board reset

def append_log(timestamp, temp, hum, status, score):
    """Add one reading to the log. Remove oldest if full."""
    entry = {
        'ts'     : timestamp,   # e.g. '09:32:15 30/05/25'
        'temp'   : temp,         # float °C or None
        'hum'    : hum,          # float % or None
        'status' : status,       # string
        'score'  : score         # 0-3
    }
    log.append(entry)
    if len(log) > MAX_LOG:
        log.pop(0)       # remove oldest entry (FIFO)

def build_json():
    """Return all log entries as a JSON string."""
    entries = []
    for e in log:
        entries.append(
            '{{"ts":"{}","temp":{},"hum":{},"status":"{}","score":{}}}'.format(
                e['ts'],
                e['temp'] if e['temp'] is not None else 'null',
                e['hum']  if e['hum']  is not None else 'null',
                e['status'], e['score']
            )
        )
    return '["' + '","'.join([]) + '"]\n' if not entries else '[' + ', '.join(entries) + ']'

        conn.close()

    except Exception as e:
        print('Error:', e)

🚨 Troubleshooting
  • RTC shows wrong time — re-run the time-set Shell code from Step 2 with correct values
  • RTC error on init — check the 4-pin Gravity cable is in an I²C port. If using DHT20 as well, use the Gravity I²C Hub — both devices need to share the same bus
  • /history shows no readings — the log only fills when you visit / (each visit triggers a read and append). Visit the dashboard a few times first.
  • I²C address conflict — RTC (0x32) and DHT20 (0x38) have different addresses. They can run on the same bus simultaneously without conflict.
  • Saved to wrong place — File → Save as → MicroPython device, not "This computer"
