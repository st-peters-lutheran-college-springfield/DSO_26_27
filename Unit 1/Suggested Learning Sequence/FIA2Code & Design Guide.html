Pico W — Step by Step Mac M4 Pathway

Your personal setup guide for the Raspberry Pi Pico W (2022, RP2040) using your MacBook. DHT22 on GP15, PIR on GP14, standalone WiFi hotspot. Follow every step in order — do not skip ahead.

🍓 Pico W 2022 · RP2040 🍎 Mac M4 — no drivers needed 🌡️ DHT22 → GP15 (pin 20) 👁 PIR → GP14 (pin 19) 📡 Hotspot AP mode ⏱ ~40 minutes
FIA2 Project Hub › Pico W Setup — student_name
🚨

Important — GPIO24 is NOT available on the Pico W

GPIO24 is used internally by the Pico W's WiFi chip (CYW43439). If you wire the PIR to GPIO24 and call Pin(24, Pin.IN), it will not work as a sensor input and may interfere with your WiFi hotspot.

Your PIR sensor goes on GP14 (physical pin 19) — the files generated for you already use Pin(14, Pin.IN). GP14 is a standard GPIO pin with no internal conflicts. If you have already wired to a different pin, update pir = Pin(14, Pin.IN) in main.py to match.

Setup steps — follow in order
1
⏱ 5 minutes

Download MicroPython firmware for the Pico W

The Pico W (2022) uses the RPI_PICO_W firmware — not Pico 2 W, not plain Pico.

  1. Go to micropython.org/download/RPI_PICO_W/ ↗
  2. Under Releases, click the first .uf2 link — latest stable version
  3. Save to your Downloads folder
⚠️

Download RPI_PICO_W only — not RPI_PICO2_W (Pico 2 W) and not RPI_PICO (no WiFi). Wrong firmware = WiFi won't work.

2
⏱ 3 minutes

Flash MicroPython onto the Pico W

  1. Locate the small BOOTSEL button on top of the board
  2. Hold BOOTSEL down with your finger
  3. While still holding, plug the micro-USB cable into your Mac — hold for 2 more seconds, then release
  4. A drive called RPI-RP2 appears on your Mac desktop (in Finder)
  5. Drag the .uf2 file from Downloads into the RPI-RP2 drive
  6. The drive disappears automatically — this means it worked ✅
✅

Drive disappears = firmware installed. If the drive stays and the file is still there, you downloaded the wrong firmware — go back to Step 1.

3
⏱ 4 minutes

Install or check Thonny — Apple Silicon (ARM64) version

  1. Go to thonny.org ↗
  2. Click the Mac download — check the filename contains arm64 (e.g. thonny-4.x.x-arm64.pkg)
  3. Open the .pkg and follow the installer
  4. If macOS says "cannot be opened because the developer cannot be verified": System Settings → Privacy & Security → Open Anyway
⚠️

If you already have Thonny, check Thonny → About Thonny — version must be 4.1 or newer and the filename must say arm64. The Intel (x86_64) build may not connect correctly on Mac M4.

4
⏱ 3 minutes

Connect Thonny to the Pico W

Plug in your Pico W normally — do NOT hold BOOTSEL this time.

  1. Open Thonny
  2. Click Run → Configure interpreter…
  3. Select MicroPython (Raspberry Pi Pico)
  4. Under Port, choose the entry containing usbmodem — e.g. /dev/cu.usbmodem1101
  5. Click OK

You should see MicroPython v1.24.x; Raspberry Pi PicoW with RP2040 and a >>> prompt in the Shell. ✅

⚠️

Port locked / Errno 35 error?

Another Thonny instance may be holding the port. Cmd+Q Thonny completely, unplug the Pico W, plug back in, reopen Thonny. To open a second Thonny for a second device: open Terminal and run:

Terminal
open -n /Applications/Thonny.app
5
⏱ 8 minutes

Wire the DHT22 and PIR sensors

Use the wiring table and pin diagram above. Both sensors connect with Gravity 3-pin cables via female-to-male Dupont jumper wires — no IO Shield needed.

SensorWirePico W pinPhysical #
DHT22 Black GNDPin 38 (or any GND)
Red 3V3 OUTPin 36
Yellow GP15Pin 20
PIR Black GNDPin 28 (or any GND)
Red 3V3 OUTPin 36 (share with DHT22)
Yellow GP14Pin 19
⚠️

Both sensors need 3V3 OUT (pin 36) for power. If both red wires need to share pin 36, use a small breadboard to split one 3V3 pin into two connections. Never use VBUS (5V, pin 40).

⏱

The PIR sensor needs ~30 seconds warm-up after first powering on. It will fire false motion detections during this time — this is normal. Wait 30 s before testing motion detection.

6
⏱ 5 minutes

Save boot.py to the Pico W

Your generated boot.py file is in the downloaded files. This handles the WiFi hotspot.

  1. In Thonny: File → Open → navigate to your downloaded boot.py
  2. Read the comments — understand what each section does before saving
  3. File → Save as…
  4. In the dialog, click "MicroPython device" (not "This computer")
  5. Filename: type exactly boot.py → click OK
  6. Press the RST button on the Pico W
  7. Watch the Thonny Shell — you should see:
Thonny Shell — expected output after RST
boot.py: starting WiFi hotspot...
========================================
Hotspot ready!
Network name : RestMonitor
Password     : rest1234
Dashboard URL: http://192.168.4.1
========================================
Connect your MacBook or tablet to "RestMonitor"
Then open http://192.168.4.1 in a browser
🚨

Naming convention — critical

Save your working backup on your MacBook as student_name_boot_v1.py. Save to the board as exactly boot.py — no prefix, no suffix. MicroPython only runs a file named exactly boot.py.

7
⏱ 5 minutes

Save main.py to the Pico W

  1. In Thonny: File → Open → navigate to your downloaded main.py
  2. Read the section comments — identify which lines are SELECTION, ITERATION, DATA OUTPUT
  3. File → Save as… → MicroPython device
  4. Filename: type exactly main.py → OK
  5. Click the green Run ▶ button
  6. Shell should show:
Thonny Shell — expected output when main.py runs
Server ready | DHT22:GP15 | PIR:GP14 | hotspot:RestMonitor
Open browser: http://192.168.4.1
... then after each browser request:
T:21.4 H:58.0 Calm:12.0s Motion:False → MONITOR CONDITIONS
8
⏱ 5 minutes

Connect your MacBook to the Pico W hotspot and open the dashboard

  1. On your MacBook: click the WiFi icon in the menu bar
  2. Find RestMonitor in the network list — click it
  3. Enter password: rest1234
  4. Open Safari, Chrome, or Firefox
  5. Type exactly: http://192.168.4.1 — include http:// not https://
  6. Daniel's dashboard should appear ✅
💡

Testing your sensors

  • DHT22: breathe gently on the sensor — temperature or humidity tile should change within 10 s
  • PIR: wave your hand near the sensor — status should change to MONITOR or NOT SUITABLE. Wait 30 s still — should return to SUITABLE
  • Shell: every browser refresh prints current readings — watch values change as you test
⚠️

While connected to RestMonitor, your MacBook has no internet. This is normal — the Pico W hotspot is a local network only. To use the internet, switch back to your normal WiFi. You can swap between them any time.

9
⏱ 5 minutes

Document in your OneNote journal — do this before the lesson ends

  1. Screenshot the Thonny Shell showing "Server ready" — paste into OneNote with today's date and duration
  2. Screenshot the working dashboard in your browser — paste into OneNote
  3. In main.py, identify and label in your journal: one line that is SELECTION, one that is ITERATION, one that is DATA OUTPUT
  4. Note the IP address (192.168.4.1) and explain in one sentence why it is always the same in AP mode
  5. Save backup copies on your MacBook: student_name_boot_v1.py and student_name_main_v1.py

Troubleshooting — Mac M4 specific
Problem Fix
RPI-RP2 drive doesn't appear after holding BOOTSELTry a different cable — must be a data micro-USB cable, not charge-only. Try a different USB-A port on your Mac or hub.
UF2 copies but RPI-RP2 drive stays visibleWrong firmware. Re-download RPI_PICO_W — not Pico 2 W, not plain Pico.
Thonny shows no port / "Could not connect" / Errno 35Cmd+Q Thonny, unplug, replug (no BOOTSEL), reopen Thonny. Select port with usbmodem.
Shell shows nothing after RST — no "Hotspot ready"boot.py may not be saved to the device. In Thonny: View → Files → check MicroPython device panel shows boot.py.
DHT22 reads None or raises exceptionCheck yellow wire is on GP15 (pin 20). Red wire on 3V3 OUT (pin 36) — not VBUS. DHT22 needs 2 s between reads — time.sleep(2) in server loop handles this.
PIR never detects / always detectsConfirm yellow wire is on GP14 (pin 19) — not GPIO24. Wait 30 s warm-up. Check code uses Pin(14, Pin.IN).
RestMonitor network not visible on MacBookConfirm main.py is running (Shell shows "Server ready"). AP_SSID in boot.py must match what you're searching for. Try turning MacBook WiFi off then on.
Browser can't load 192.168.4.1MacBook must be connected to RestMonitor (not school WiFi). Type http:// not https://. Use exact address — no www.
