Lesson 11 — Thursday 13 August 2026
Week B, Day 9 · Period 2 (60 min) · Objective(s) 6
Subject matter
  • SQL syntax; CREATE, INSERT and DELETE statements
  • Data types, constraints, and primary/foreign keys
Learning experiences
  • Warm-up: explore a working relational database first — open the sample music database in DB Browser for SQLite, inspect its CREATE TABLE SQL and run a provided SELECT query
  • Build your own SQLite database: use the Python Database Tool to create a new .db file, then write and run your CREATE TABLE devices and CREATE TABLE events statements (PK/FK, data-type constraints) directly in its SQL box
  • Transfer your working SQL into a validated Python program: write INSERT statements with Python validation (reject nulls, constrain classification values) using the sqlite3 module
Resources
  • DB Browser for SQLite + sample database — Database Essentials Starter Pack
  • Python Database Tool — setup & worked example guide
  • Python sqlite3 module docs
  • w3schools SQL tutorial (create, insert)
  • Laptop / VS Code

Lesson 12 — Friday 14 August 2026
Week B, Day 10 · Period 4 (60 min) · Objective(s) 6
Subject matter
  • SELECT, WHERE, GROUP BY, HAVING, ORDER BY, COUNT, inner-joins and sub-queries
Learning experiences
  • Optional: prototype each SELECT/JOIN/GROUP BY query in the Python Database Tool first to check syntax and output before wiring the query string into your program
  • Write SELECT queries for Jordan Kim's three use cases (device filter; date/classification filter; GROUP BY/COUNT summary)
  • Write a JOIN query linking events to devices; test all SQL against a sample dataset
Resources
  • Python Database Tool — setup & worked example guide
  • w3schools SQL tutorial (select, joins, group by)
  • Sample test dataset
