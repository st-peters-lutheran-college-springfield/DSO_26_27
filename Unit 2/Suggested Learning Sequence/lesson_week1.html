Week 1: Two-Tier System & Database Foundations

Part A (Project Learning — Competency-Based) · Lessons 1–4 · Tuesday 28 July 2026 – Friday 31 July 2026

Four 60-minute lessonsNo checkpoint this week
Start here: work through the four lessons below in order. Log evidence in your OneNote journal as you go — don’t leave it until the end of the week.

This Week’s Lessons

Lesson 1 — Tuesday 28 July 2026
Week B, Day 7 · Period 3 (60 min) · Objective(s) 1
Subject matter
  • Nature of data-driven problems; scope, constraints and limitations of environments
  • Elements needed for a data-driven solution: platforms, connections, hardware, data stores
Learning experiences
  • Unit and assessment overview: introduce the Sunrise Aged Care scenario and the two-part Technical Proposal (Part A build → Part B propose)
  • Introduce the two-tier system architecture: ESP32-S3 edge device + laptop SQLite backend
  • Unbox and assemble the ESP32-S3 AI Chatbot Kit on breadboard; run display "hello world"
  • Start OneNote project journal — first timestamped entry
Resources
  • ESP32-S3 AI Chatbot Kit (KS-KS5027)
  • Keyestudio assembly tutorial
  • Unit 2 assessment instrument (Part A + B)
  • OneNote

Lesson 2 — Wednesday 29 July 2026
Week B, Day 8 · Period 5 (60 min) · Objective(s) 1
Subject matter
  • Recognise data types, constraints, primary and foreign keys
  • Relations (tables): rows, columns, PK/FK, nulls
Learning experiences
  • Starter: complete the Database Key Terms Quiz (table, field, record, primary key) using the sample music database — a quick vocabulary check before applying the terms to your own data
  • Direct teaching: data types, PK/FK constraints, relational table structure
  • Examine the PROVIDED devices/events database (supplied in 2NF)
  • Identify the normalisation anomaly in the provided structure
Resources
  • Database Key Terms Quiz — Database Essentials Starter Pack
  • Provided 2NF database file (devices, events)
  • w3schools SQL tutorial
  • Worked examples

Lesson 3 — Thursday 30 July 2026
Week B, Day 9 · Period 2 (60 min) · Objective(s) 1
Subject matter
  • Data principles: organisation, representation, integrity, anomalies, redundancy
  • Difference between data, information and wisdom
Learning experiences
  • Complete the Inefficient Flat File Database worksheet first — a worked music-database example — before analysing redundancy in the provided devices/events structure
  • Normalisation workshop: identify redundancy/integrity issues in the provided database
  • Begin ER diagram for the 2NF structure (devices, events)
Resources
  • Inefficient Flat File Database worksheet — Database Essentials Starter Pack
  • ERDPlus
  • Essential SQL — Database normalization explained in simple English
  • BeginnersBook — Normalization in DBMS

Lesson 4 — Friday 31 July 2026
Week B, Day 10 · Period 4 (60 min) · Objective(s) 2
Subject matter
  • Symbolise data flow through a system using data flow diagrams
  • Elements needed for a data-driven solution: real-world data sources
Learning experiences
  • Introduce Gane-Sarson DFD notation — T (transform) and D (data store) distinction
  • Draw Level 0 DFD for the Sunrise Aged Care system (ESP32-S3 → backend → database → query interface)
Resources
  • Visual Paradigm — Gane-Sarson DFD Tutorial
  • "How to Draw Data Flow Diagram?" (YouTube)

Log Your Evidence
